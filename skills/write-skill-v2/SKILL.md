# Write-Skill V2.0 写作方法论

## 版本演进

**V1.0** → 高启示率、情感钩子、口语化表达
**V2.0** → 在 V1.0 基础上增加：信息密度控制、结构化拆解、记忆点设计

## 核心升级点

### 1. 标题公式升级

**V1.0**: 痛点/悬念 + 数字/对比
- 例："22岁开出5万美元支票：没钱也能做天使投资？"

**V2.0**: 痛点/悬念 + 数字/对比 + 价值承诺/身份标签
- 例："DeepSeek V4 发布，全网最细解读 & 技术报告拆解"
- 关键升级：明确告诉读者"你能得到什么"（最细解读、技术报告拆解）

### 2. 开篇钩子升级

**V1.0**: 个人故事/场景切入
- 优势：亲切、易读
- 劣势：信息密度低，需要读一段才知道主题

**V2.0**: 3秒法则 - 第一句话必须包含核心信息
```
❌ V1.0: "前两天刷 GitHub Trending，看到一个 repo 涨得飞快..."
✅ V2.0: "昨晚，DeepSeek 悄悄放出了 V4 预览版。百万上下文，1元/百万token。"
```

**公式**: 时间/场景 + 核心事实 + 关键数字

### 3. 信息密度控制

**V1.0 问题**: 段落较长，读者容易疲劳
**V2.0 改进**: 
- 每段不超过 3 句话
- 关键信息单独成段
- 使用视觉分隔（引用块、数字标签）

```
V2.0 段落结构:
[钩子句] - 引入话题
[事实句] - 核心数据
[解读句] - 价值判断
```

### 4. 结构化拆解

**V1.0**: 线性叙述，按时间/流程组织
**V2.0**: 模块化拆解，按"问题-方案-验证"组织

**推荐结构**:
```
01 现象/问题（为什么重要）
02 核心数据（事实是什么）
03 技术原理（怎么做到的）
04 实测场景（验证效果）
05 局限/风险（客观评价）
06 使用指南（怎么开始）
```

### 5. 记忆点设计

**V1.0**: 依赖读者自己总结
**V2.0**: 主动植入记忆点

**方法**:
- 每节一个"金句"（加粗 + 引用块）
- 数字标签（01/02/03）
- 对比表格
- 步骤清单

```
记忆点示例:
"这不是简单的版本迭代，是国产大模型第一次在长文本领域正面硬刚 GPT-4 和 Claude。"

"长文本不是目的，是基础设施。"
```

### 6. 视觉层次优化

**V2.0 新增**:
- 章节数字（01/02/03）- 80px 大号
- 引用块背景色区分（重要观点）
- 代码/数据单独区块
- 对比表格

## V2.0 写作流程

### Step 1: 信息收集（10分钟）
- 提取核心数据（数字、价格、性能指标）
- 找出 3-5 个关键事实
- 识别目标读者痛点

### Step 2: 结构设计（5分钟）
- 确定 5-7 个章节
- 每章分配一个核心信息点
- 设计章节之间的逻辑流

### Step 3: 内容填充（30分钟）
- 每段遵循"钩子-事实-解读"结构
- 每节一个金句
- 插入对比表格或清单

### Step 4: 记忆点植入（10分钟）
- 检查每章是否有可传播的金句
- 确保关键数字被强调
- 添加视觉分隔

### Step 5: 优化润色（10分钟）
- 删减冗余表述
- 检查信息密度（每段<3句）
- 验证逻辑流畅性

## V2.0 检查清单

- [ ] 标题包含"价值承诺"
- [ ] 开篇3秒内给出核心信息
- [ ] 每段不超过3句话
- [ ] 每节一个金句（加粗+引用块）
- [ ] 使用01/02/03章节编号
- [ ] 包含至少1个对比表格
- [ ] 包含至少1个步骤清单
- [ ] 结尾有明确的行动指南
- [ ] 全文信息密度适中（不累读）

## V2.0 vs V1.0 对比示例

### 标题
```
V1.0: "22岁开出5万美元支票：没钱也能做天使投资？"
V2.0: "22岁天使投资人揭秘：5万美元起投，0资金门槛的3条路径（附实操指南）"
```

### 开篇
```
V1.0:
"22 岁那年，我开出了一张 5 万美元的支票。
不是投给什么知名基金，而是投给了一个在 MIT 默默做项目的同学..."

V2.0:
"22岁，5万美元，第一笔天使投资。
不是富二代，没有家族资金，我用的是一个大多数人不知道的系统。
3年后，这笔投资被贝索斯收购。"
```

### 信息呈现
```
V1.0:
"缓存读取成本是基础输入价格的 0.1 倍——每个缓存 token 打 9 折。缓存写入成本 1.25 倍，存储 KV 张量有 25% 溢价。延长一小时缓存成本 2.0 倍。"

V2.0:
"定价策略很激进：
- 缓存读取：0.1x（打9折）
- 缓存写入：1.25x（25%溢价）
- 延长1小时：2.0x

对比 Claude 3 的 200K 上下文（约15元/百万token），DeepSeek-V4 便宜了一个数量级。"
```

## 适用场景

- 技术产品发布解读
- 行业趋势分析
- 工具/方法论教程
- 投资/商业分析

## 不适用场景

- 情感故事（用卡兹克风格）
- 纯新闻报道（用标准客观风格）
- 学术论文（用专业严谨风格）

## 版本选择指南

| 场景 | 推荐版本 |
|------|----------|
| 个人成长/创业故事 | V1.0 或 卡兹克风格 |
| 技术产品解读 | V2.0 |
| 行业分析报告 | V2.0 |
| 工具教程 | V2.0 |
| 热点快评 | V1.0 |
